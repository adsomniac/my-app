Isi dari planning-nya adalah sebagai berikut:
# ISSUE: Implementasi Authentication & Role-Based Authorization dengan JWT

## 📌 Deskripsi

Implementasikan sistem Authentication dan Authorization untuk aplikasi menggunakan:

* Backend: Golang `net/http`
* Frontend: Svelte 5
* Database: PostgreSQL
* Authentication: JWT
* Password Hashing: bcrypt
* Authorization: Role-Based Access Control (RBAC)

Sistem memiliki tiga role:

```text
user
admin
super_admin
```

Setelah berhasil login, user diarahkan ke halaman sesuai role.

| Role          | Dashboard     |
| ------------- | ------------- |
| `user`        | `/dashboard`  |
| `admin`       | `/admin`      |
| `super_admin` | `/superadmin` |

## Tanggung Jawab Layer

### Controller

Bertanggung jawab terhadap:

* HTTP Request
* Decode JSON
* Validasi input dasar
* Memanggil Service
* HTTP Status Code
* JSON Response

Tidak boleh menjadi tempat semua logic bisnis.

---

### Service

Bertanggung jawab terhadap:

* Business logic
* Validasi credential
* Verifikasi password
* Pemeriksaan user aktif
* Generate JWT

Service tidak boleh mengetahui detail HTTP.

Tidak boleh menggunakan:

```go
http.ResponseWriter
```

atau:

```go
json.NewEncoder(w)
```

---

### Repository

Bertanggung jawab terhadap:

* Query database
* Insert data
* Update data
* Mengambil data dari PostgreSQL

Repository tidak boleh:

* Generate JWT
* Mengatur HTTP Response
* Membaca Authorization Header

---

### Middleware

Bertanggung jawab terhadap:

* Membaca JWT dari request
* Validasi token
* Mengecek expiration
* Mengambil informasi user
* Menyimpan informasi user ke `context.Context`

# PHASE 1 — Database Migration

## File

```text
database/migrations/001_create_users_table.sql
```

Buat tabel:

```sql
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,

    email VARCHAR(255) NOT NULL,

    password_hash TEXT NOT NULL,

    role VARCHAR(20) NOT NULL DEFAULT 'user',

    is_active BOOLEAN NOT NULL DEFAULT TRUE,

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT users_role_check
        CHECK (
            role IN (
                'user',
                'admin',
                'super_admin'
            )
        )
);
```

Buat unique index untuk email:

```sql
CREATE UNIQUE INDEX users_email_unique
ON users (LOWER(email));
```

## Tujuan

Database harus mencegah:

```text
Role tidak valid
Email duplikat
User tanpa password
User tanpa email
```

## Acceptance Criteria

* [ ] Tabel `users` berhasil dibuat
* [ ] `id` menggunakan auto increment
* [ ] `email` wajib diisi
* [ ] Email tidak boleh duplikat
* [ ] Email bersifat case-insensitive
* [ ] `password_hash` wajib diisi
* [ ] Role default adalah `user`
* [ ] Role hanya boleh:

  * [ ] `user`
  * [ ] `admin`
  * [ ] `super_admin`
* [ ] `is_active` default `true`
* [ ] Timestamp menggunakan `TIMESTAMPTZ`

# PHASE 5 — Password Hashing dengan bcrypt

Password tidak boleh disimpan sebagai plaintext.

Salah:

```text
password123
```

Benar:

```text
$2a$10$xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## Testing Development

Untuk kebutuhan development/testing, implementor diperbolehkan menggunakan third-party bcrypt generator untuk membuat bcrypt hash.

Contoh:

```text
Password:
password123

Bcrypt Hash:
$2a$10$...
```

Kemudian:

```sql
INSERT INTO users (
    email,
    password_hash,
    role
)
VALUES (
    'admin@test.com',
    '<BCRYPT_HASH>',
    'admin'
);
```

## Aturan

Third-party bcrypt generator hanya digunakan untuk:

```text
Development
Testing
Dummy Seed Data
```

Jangan gunakan:

* Password asli user
* Password production
* Password administrator sebenarnya
* Password yang digunakan pada sistem lain

Jangan commit plaintext password ke Git.

Untuk testing lokal, gunakan password dummy.

Contoh:

```text
Email:
admin@test.com

Password:
password123
```


```go

```

Method:

```go

    ctx context.Context,
    email string,
    password string,
) (*models.LoginResult, error)
```

Flow:

```text
Email & Password
       ↓
Find User By Email
       ↓
User ditemukan?
       │
       ├── Tidak → Invalid Credential
       │
       └── Ya
            ↓
       Is Active?
            │
            ├── Tidak → Reject
            │
            └── Ya
                 ↓
       Compare Password
                 ↓
           Password valid?
                 │
                 ├── Tidak → Invalid Credential
                 │
                 └── Ya
                      ↓
                Generate JWT
                      ↓
                Return Result
```

Password verification:

```go
bcrypt.CompareHashAndPassword(
    []byte(user.PasswordHash),
    []byte(password),
)
```

## Catatan

Error berikut harus menghasilkan response login yang sama:

```text
User tidak ditemukan
Password salah
```

Response:

```text
Email atau password salah
```

# PHASE 11 — Role-Based Authorization

Buat middleware untuk authorization.

Rekomendasi:

```go
RequireMinimumRole("admin")
```

Role hierarchy:

```text
user        = level 1
admin       = level 2
super_admin = level 3
```

Permission:

| Required Role | user | admin | super_admin |
| ------------- | ---- | ----- | ----------- |
| user          | ✅    | ✅     | ✅           |
| admin         | ❌    | ✅     | ✅           |
| super_admin   | ❌    | ❌     | ✅           |

Contoh:

```text
GET /api/admin/users
        ↓
AuthMiddleware
        ↓
RequireMinimumRole("admin")
        ↓
Controller
```

Jika role tidak memenuhi:

```text
403 Forbidden
```

Penting:

> Frontend boleh menyembunyikan menu berdasarkan role, tetapi backend tetap wajib melakukan authorization.

# PHASE 20 — Logout

Flow:

```text
Dashboard
    ↓
Logout
    ↓
Hapus auth_token
    ↓
Authentication State = false
    ↓
Login Page
```

Implementasi:

```javascript
localStorage.removeItem("auth_token");
```

## Catatan JWT

JWT bersifat stateless.

Artinya:

```text
User Logout
    ↓
Token dihapus dari browser
```

Tetapi token yang sebelumnya dicuri secara teori masih valid sampai:

```text
exp
```

Untuk scope fitur ini:

```text
Client-side logout
```

sudah cukup.

Token revocation/blacklist tidak termasuk scope.


Jelaskan tahapan-tahapan yang harus dilakukan untuk mengimplementasikan fitur ini, anggap nanti yang mengimplementasikan adalah junior programmer atau model AI yang lebih murah.
